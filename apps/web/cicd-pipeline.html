CI/CD Pipeline

This page documents how code moves from GitHub into the SchoolCloud AWS environment, including Terraform automation, security checks, and deployment history.

GitHub Actions Terraform OIDC to AWS Security Scans

CI/CD Architecture

Developers push changes to GitHub. GitHub Actions runs Terraform plans on pull requests and full Terraform applies on the main branch, then syncs the web app to S3 and invalidates CloudFront. Planned additions include tfsec/checkov scans and deployment summaries stored in S3.

Pipeline flow (today + planned)

  1. Developer opens a pull request to the repo.
  2. Terraform Plan workflow runs on PR: checkout, setup Terraform, configure AWS (OIDC), terraform fmt, validate, plan.
  3. (Planned) Security scanners run: tfsec, checkov, maybe npm test / npm audit.
  4. After review, the PR is merged to main.
  5. Deploy workflow runs on main: Terraform init/plan/apply, then syncs apps/web to the S3 web bucket and triggers a CloudFront invalidation.
  6. (Planned) A small step writes a deployment-summary.json to an S3 bucket describing the build, commit, and which checks passed.
  7. (Planned) A Deployment Summary Lambda reads the last few JSON files and exposes them to this portal as “Last 5 deployments” with green/red status.
