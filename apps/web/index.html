Secure access for students, teachers, and admins.

The SchoolCloud Portal uses AWS Cognito, API Gateway, Lambda, and WAF to give schools a simple, auditable way for users to manage their profiles and view their own activity.

Students – update your profile and see your activity. Teachers – verify access and keep accounts secure. Admins – monitor security events and alerts.

About the project

This portal was built as a secure serverless reference for K–12 and higher education. It focuses on strong identity, logging, and security visibility.

Highlights:

  • Serverless architecture (CloudFront, S3, API Gateway, Lambda)
  • Per-user profiles with role (student / teacher / admin)
  • Activity page that surfaces CloudTrail & audit events
  • EventBridge + SNS alerts for sensitive actions

Architecture Overview

The SchoolCloud Portal runs entirely on AWS managed services. Static content is served from S3 through CloudFront and AWS WAF, while API Gateway and Lambda handle profile and activity logic backed by DynamoDB and CloudTrail.
